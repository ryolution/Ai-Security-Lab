# Code of Conduct

AI Security Lab follows a simple contributor standard: be respectful, precise, and constructive.

Security work can involve sensitive systems and stressful findings. Keep discussions focused on reproducibility, evidence, impact, and remediation. Do not harass contributors, publish private data, or pressure maintainers to disclose vulnerability details before a safe path exists.

Maintainers may remove content or participation that puts people, systems, or the project at risk.
